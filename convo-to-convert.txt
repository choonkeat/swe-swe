● Now I have the full picture. Here's the root cause:

  Each terminal-ui element on the page independently connects a debug
  WebSocket to /__agent-reverse-proxy-debug__/ui (line 3487 of
  terminal-ui.js). When you have 2 sessions visible (e.g., an agent
  session + a shell session), both register as UI observers with the
  reverse proxy's DebugHub.

  When you run open http://example.com:

  1. The swe-swe-open shim curls
  /__agent-reverse-proxy-debug__/open?url=http://example.com
  2. The reverse proxy calls hub.SendToUIObservers(msg) which broadcasts
   { t: "open", url: "..." } to all connected UI observer WebSockets
  (debughub.go:124-137)
  3. Each terminal-ui element receives the message and calls
  this.openIframePane('preview', msg.url) (terminal-ui.js:3527-3529)
  4. openIframePane → setPreviewURL detects example.com is external and
  shows confirm('Open in new tab?\n\n' + targetURL)
  (terminal-ui.js:3364-3370)

  So you get one confirm() dialog per connected terminal session. With 2
   sessions, you get 2 prompts.

  Fix options:
  - In terminal-ui.js: only handle { t: 'open' } on the focused/active
  terminal session, or deduplicate with a timestamp
  - In the reverse proxy: send open to only one UI observer instead of
  broadcasting

  same endpoint
